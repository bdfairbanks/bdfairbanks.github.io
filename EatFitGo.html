You Need Cultivators...

People who believe in the vision and growth of Eat Fit Go. I fit that bill. I also bring to the table a general flexibility that allows me to dream up innovative solutions to problems that inevitably arise. This is augmented by my commitment to segmented, understandable code. Preventing code sprawl has been a large part of my development as a coder, so some section of my thought is always focused on creating more efficient passages. Noticing the marked improvements this has brought in both the functionality and understandability of my code pushed me towards a breakthrough: Code is not only composed of procedural languages, but is a language in and of itself, one that is most fluent when spoken with preciseness and brevity.

How this can help

Good back end code has a logic to it. A fires B which sets the stage for C which triggers D at just the right point. Getting to the point where it is this simple to explain your code out loud is a victory in and of itself for me. A structurally sound back end removes impediments for front end coding, allowing it to become an exercise of creativity, rather than a minefield of potential conflicts.
