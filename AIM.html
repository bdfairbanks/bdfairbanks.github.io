How this can help

Good code has a logic to it. A fires B which sets the stage for C which triggers D at just the right point. Getting to the point where it is this simple to explain your code out loud is a victory in and of itself for me. Testing is an augmentation of this mantra. Every stage of testing is helpful to the code refinement process. The process of desigining a test almost always shows if a method is doing too much, the test set up allows you to find the precise limits of what your code can do and executing the tests can suggust potential uses for the method that perhaps where not apperent before.
